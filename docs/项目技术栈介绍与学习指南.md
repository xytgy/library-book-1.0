# 图书馆管理系统 — 技术栈介绍与学习指南

## 一、项目简介

这是一个**纯 Java SE** 实现的控制台图书馆管理系统，不依赖任何 Web 框架（如 Spring Boot）。通过 JDBC 直接操作 MySQL 数据库，使用数字菜单驱动的终端界面进行交互。

**核心价值**：理解 Java 基础、JDBC 原理、手动事务管理、密码加密、JWT 认证等底层机制。

### 2.2 JDBC（数据库连接）

**是什么**：Java Database Connectivity，Java 标准的数据库访问 API。通过 JDBC 可以用 Java 代码执行 SQL 语句。

**为什么用**：直接操作数据库，理解数据库交互的本质。

**核心类**：
| 类 | 作用 | 本项目位置 |
|---|------|-----------|
| `DriverManager` | 获取数据库连接 | `DatabaseUtil.java` |
| `Connection` | 数据库连接对象 | 所有 DAO 类 |
| `PreparedStatement` | 预编译 SQL 语句 | 所有 DAO 类 |
| `ResultSet` | 查询结果集 | 所有 DAO 类 |

**典型用法**（以 BookDao.findById 为例）：
```java
public Book findById(Long id) {
    String sql = "SELECT * FROM books WHERE id = ?";
    Connection conn = null;
    PreparedStatement ps = null;
    ResultSet rs = null;
    try {
        conn = DatabaseUtil.getConnection();    // 1. 获取连接
        ps = conn.prepareStatement(sql);        // 2. 预编译 SQL
        ps.setLong(1, id);                       // 3. 设置参数（防 SQL 注入）
        rs = ps.executeQuery();                  // 4. 执行查询
        if (rs.next()) return mapRow(rs);        // 5. 处理结果
    } catch (SQLException e) {
        throw new RuntimeException("查询失败", e);
    } finally {
        DatabaseUtil.close(conn, ps, rs);       // 6. 关闭资源
    }
    return null;
}
```

**关键概念**：
- **PreparedStatement** vs Statement：PreparedStatement 预编译 SQL，防止 SQL 注入
- **参数化查询**：用 `?` 占位符代替字符串拼接
- **资源关闭**：必须在 finally 或 try-with-resources 中关闭连接

### 2.3 HikariCP（连接池）

**是什么**：高性能 JDBC 连接池。数据库连接创建开销大，连接池预先创建一批连接复用，避免每次都新建。

**为什么用**：每次 JDBC 操作都需要获取/关闭连接，连接池显著提升性能。

**配置**（`application.properties`）：
```properties
db.pool.maxSize=10          # 最大连接数
db.pool.minIdle=5           # 最小空闲连接
db.pool.connectionTimeout=20000  # 获取连接超时时间（ms）
db.pool.idleTimeout=300000       # 空闲连接超时时间（ms）
```

**本项目使用**：
```java
// DatabaseUtil.java
private static HikariDataSource dataSource;

public static void init() {
    HikariConfig config = new HikariConfig();
    config.setJdbcUrl(url);
    config.setUsername(user);
    config.setPassword(password);
    dataSource = new HikariDataSource(config);  // 创建连接池
}

public static Connection getConnection() throws SQLException {
    return dataSource.getConnection();  // 从池中获取连接
}
```

**关键 SQL 操作**：

### 2.5 BCrypt（密码加密）

**是什么**：一种密码哈希算法，专门用于安全地存储密码。每次加密结果不同（加盐），无法逆向破解。

**为什么用**：密码不能明文存储，BCrypt 是业界标准。

**本项目使用**：
```java
// 注册时：加密密码
String hash = BCrypt.withDefaults().hashToString(12, password.toCharArray());
// 结果类似：$2a$10$nsBRs8RxEo260Aes4t833.NMlwc/ZVoMusr835aYJPJcFIpxVr0KK

// 登录时：验证密码
BCrypt.Result result = BCrypt.verifyer().verify(
    inputPassword.toCharArray(),   // 用户输入的密码
    storedHash                     // 数据库中的哈希
);
if (result.verified) {
    // 密码正确
}
```

**注意**：`$2a$` 前缀是 BCrypt 的标准格式，`10` 是 cost factor（计算强度）。

### 2.6 JWT（JSON Web Token）

**是什么**：一种无状态的认证令牌。登录后服务端生成 Token 返回给客户端，后续请求携带 Token 即可证明身份。

**为什么用**：无状态，不需要 Session 存储，适合前后端分离。

**Token 结构**：
```
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOjEsInVzZXJuYW1lIjoiYWRtaW4iLCJyb2xlIjoiYWRtaW4ifQ.signature
│                    │ │                                                         │
│   Header           │ │                      Payload                           │ │ Signature
│  (算法信息)         │ │              (用户信息：ID、用户名、角色)                  │ │  (签名)
```

**本项目使用**：
```java
// 生成 Token
String token = jwtUtil.generateToken(userId, username, role);

// 解析 Token
Long userId = jwtUtil.getUserId(token);
String username = jwtUtil.getUsername(token);
String role = jwtUtil.getRole(token);

// 验证 Token
boolean valid = jwtUtil.validateToken(token);
```

**注意**：本项目是控制台应用，JWT 主要用于理解 Token 认证原理。

## 六、关键设计模式

### 6.1 手动事务管理（BorrowService）
```java
Connection conn = DatabaseUtil.getConnection();
conn.setAutoCommit(false);           // 开启事务
try {
    // 多个 SQL 操作...
    conn.commit();                    // 提交事务
} catch (Exception e) {
    conn.rollback();                  // 回滚事务
    throw e;
} finally {
    conn.close();                     // 关闭连接
}
```

### 6.2 悲观锁（SELECT FOR UPDATE）
```java
// 锁定行，其他事务必须等待
Book book = bookDao.findByIdForUpdate(conn, bookId);
// 此时其他线程执行 SELECT * FROM books WHERE id = bookId 会被阻塞
// 直到当前事务 commit 或 rollback
```

### 6.3 分层架构
```
Menu（交互） → Service（业务） → DAO（数据） → Database（存储）
```
每层只依赖下一层，职责清晰，易于测试和维护。

## 七、常见问题

### Q: 为什么不直接用 Spring Boot？
A: 课程要求纯 Java SE，帮助理解 JDBC 原理、手动事务管理、依赖注入本质等底层机制。

### Q: SELECT FOR UPDATE 是什么？
A: 悲观锁。执行查询时锁定数据库行，其他事务必须等待锁释放。用于防止并发借书时的超卖问题。

### Q: BCrypt 为什么每次加密结果不同？
A: BCrypt 自动加盐（salt），每次生成不同的随机盐值，即使密码相同哈希也不同，防止彩虹表攻击。

### Q: JWT Token 为什么不需要存数据库？
A: JWT 是自包含的，Token 里就包含用户信息。服务端只需要验证签名是否有效，不需要查询数据库，所以是"无状态"的。
