# 图书馆管理系统 — 6人团队分工计划

---

## 人员1：数据库与模型工程师

**角色定位**：负责整个项目的数据基础，包括数据库设计和所有实体类。

### 负责文件（4个）

#### 1. `database/init.sql`
- **功能**：数据库初始化脚本，创建所有表并插入测试数据
- **内容**：
  - 创建 `users` 表（id, username, password, name, email, role, status, created_at, updated_at）
  - 创建 `books` 表（id, isbn, title, author, category, location, total_quantity, available_quantity, description, created_at, updated_at）
  - 创建 `borrow_records` 表（id, user_id, book_id, borrow_date, due_date, return_date, status, created_at, updated_at）
  - 外键约束：borrow_records.user_id → users.id，borrow_records.book_id → books.id
  - 索引：user_id、book_id、status
  - 测试数据：admin/user1 账号 + 5本书
- **如何上手**：
  1. 先理解三张表的关系：用户借书 → 产生借阅记录
  2. 注意字段类型：password 用 VARCHAR(255)（BCrypt哈希很长），role 用 ENUM
  3. 密码哈希格式必须是 `$2a$10$` 开头（BCrypt 标准）
  4. 执行命令测试：`mysql -u root -p123456 < database/init.sql`

#### 2. `model/User.java`
- **功能**：用户实体类，对应 users 表
- **字段**：id(Long), username(String), password(String), name(String), email(String), role(String), status(Integer), createdAt(LocalDateTime), updatedAt(LocalDateTime)
- **如何上手**：
  1. 字段名用驼峰命名（Java），数据库列名用下划线（如 created_at → createdAt）
  2. 提供无参构造方法 + 所有 getter/setter

#### 3. `model/Book.java`
- **功能**：书籍实体类，对应 books 表
- **字段**：id, isbn, title, author, category, location, totalQuantity, availableQuantity, description, createdAt, updatedAt
- **如何上手**：同 User.java，注意 totalQuantity/availableQuantity 是 Integer 类型

#### 4. `model/BorrowRecord.java`
- **功能**：借阅记录实体类，对应 borrow_records 表
- **字段**：id, userId, bookId, borrowDate, dueDate, returnDate, status, createdAt, updatedAt
- **额外字段**（联表查询用）：bookTitle(String), bookAuthor(String), userName(String)
- **如何上手**：
  1. 基本字段对应 borrow_records 表
  2. 额外的 bookTitle/bookAuthor/userName 是联表查询时从 books 和 users 表带出来的，用于显示

## 人员2：用户管理工程师

**角色定位**：负责用户相关的完整功能链路：数据访问 → 业务逻辑 → 界面交互。

### 负责文件（4个）

#### 1. `dao/UserDao.java`
- **功能**：用户数据访问层，所有用户相关的数据库操作
- **核心方法**：
  - `findById(Long id)` — 根据ID查询用户
  - `findByUsername(String username)` — 根据用户名查询（登录用）
  - `existsByUsername(String username)` — 检查用户名是否存在（注册用）
  - `save(User user)` — 插入新用户，返回自增ID
  - `update(User user)` — 更新用户信息
  - `deleteById(Long id)` — 删除用户
  - `findAll(String search, int page, int pageSize)` — 分页查询+搜索
  - `count(String search)` — 统计总数（分页用）
- **如何上手**：
  1. 每个方法模板：获取连接 → 预编译SQL → 设置参数 → 执行 → 处理结果 → 关闭资源
  2. 用 try-finally 确保资源关闭：`DatabaseUtil.close(conn, ps, rs)`
  3. 搜索用 LIKE 模糊查询：`WHERE username LIKE ? OR name LIKE ? OR email LIKE ?`
  4. 分页用 LIMIT + OFFSET：`LIMIT ? OFFSET ?`，OFFSET = (page-1) * pageSize
  5. save 方法需要返回自增ID：`ps.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS)`

#### 2. `service/UserService.java`
- **功能**：用户业务逻辑层
- **核心方法**：
  - `getUsers(search, page, pageSize)` — 分页查询用户列表
  - `getUserById(id)` — 查询用户详情（不存在则抛异常）
  - `updateUser(id, name, email, role, status, currentUserId, currentUserRole)` — 更新用户（含权限校验）
  - `deleteUser(id)` — 删除用户（需检查是否有未归还借阅）
- **如何上手**：
  1. 先注入依赖：`UserDao userDao` + `BorrowRecordDao borrowRecordDao`
  2. update 权限逻辑：只有 admin 或本人可修改
  3. delete 前检查：`borrowRecordDao.existsActiveBorrowByUserId(id)` 有未还书则拒绝删除
  4. 不存在的记录统一抛 `BusinessException(404, "用户不存在")`

#### 3. `console/AuthMenu.java`
- **功能**：登录和注册的控制台交互
- **核心方法**：
  - `showLoginMenu()` — 显示登录界面，读取用户名密码，调用 AuthService，返回登录结果
  - `showRegisterMenu()` — 显示注册界面，读取注册信息，调用 AuthService，返回注册结果
- **如何上手**：
  1. 用 `ConsoleHelper.readString("提示", true)` 读取输入
  2. 用 `ConsoleHelper.printSuccess/Error()` 显示结果
  3. 捕获 `BusinessException` 显示错误信息
  4. 登录/注册成功返回 `Map<String, Object>`（含 token 和 user 信息）

#### 4. `util/JwtUtil.java`
- **功能**：JWT 令牌工具类
- **核心方法**：
  - `generateToken(userId, username, role)` — 生成 JWT Token
  - `parseToken(token)` — 解析 Token 获取 Claims
  - `validateToken(token)` — 验证 Token 是否有效
  - `getUserId(token)` / `getUsername(token)` / `getRole(token)` — 从 Token 提取信息
- **如何上手**：
  1. 从 `application.properties` 读取 jwt.secret 和 jwt.expiration
  2. 用 `Keys.hmacShaKeyFor()` 生成密钥
  3. 用 Jwts.builder() 构建 Token，设置 claims（userId, username, role）
  4. 用 Jwts.parser() 解析 Token

## 人员3：书籍管理工程师

**角色定位**：负责书籍相关的完整功能链路。

### 负责文件（3个）

#### 1. `dao/BookDao.java`
- **功能**：书籍数据访问层
- **核心方法**：
  - `findById(Long id)` — 查询单本书
  - `findByIdForUpdate(Connection conn, Long id)` — **悲观锁查询**（事务内用）
  - `existsByIsbn(String isbn)` — ISBN 唯一性检查
  - `save(Book book)` — 插入书籍
  - `update(Book book)` — 更新书籍
  - `deleteById(Long id)` — 删除书籍
  - `decreaseAvailable(Long bookId)` — 扣减库存（SQL: `WHERE available_quantity > 0`）
  - `increaseAvailable(Long bookId)` — 增加库存
  - `findAll(search, category, page, pageSize)` — 分页+搜索+分类筛选
  - `count(search, category)` — 统计总数
- **如何上手**：
  1. `findByIdForUpdate` 是并发安全的关键，SQL 加 `FOR UPDATE` 后缀
  2. `decreaseAvailable` 的 SQL 有保护：`WHERE available_quantity > 0` 防止负数
  3. 搜索支持按 title/author/isbn 模糊匹配
  4. 分类筛选：`WHERE category = ?`

#### 2. `service/BookService.java`
- **功能**：书籍业务逻辑
- **核心方法**：
  - `getBooks(search, category, page, pageSize)` — 分页查询
  - `getBookById(id)` — 查询详情（不存在抛异常）
  - `createBook(...)` — 创建书籍（含 ISBN 唯一性校验）
  - `updateBook(id, ...)` — 更新书籍（含 ISBN 校验 + 库存联动）
  - `deleteBook(id)` — 删除书籍（需检查是否有未归还借阅）
- **如何上手**：
  1. **库存联动**：更新 totalQuantity 时，availableQuantity 要同步调整
     - 例：total 从 10 改为 7，available 也要减 3（但不能低于 0）
  2. ISBN 唯一性：创建时检查是否存在，更新时排除自身
  3. 删除前检查：`borrowRecordDao.existsActiveBorrowByBookId(id)`

#### 3. `console/BookMenu.java`
- **功能**：书籍管理的控制台菜单
- **核心方法**：
  - `showMenu(isAdmin)` — 主菜单路由（普通用户只看列表，管理员可增删改）
  - `listBooks(isSearch)` — 书籍列表（带分页、搜索）
  - `showBookDetail()` — 查看详情
  - `createBook()` — 添加书籍
  - `updateBook()` — 编辑书籍
  - `deleteBook()` — 删除书籍（含确认提示）
- **如何上手**：
  1. 分页交互：按 P 上一页，N 下一页，Q 返回
  2. 单页数据时直接显示"按回车键返回"，不弹选择提示
  3. 编辑时显示当前值，用户回车跳过表示不修改
  4. 删除前要确认：`"确认删除? (y/n)"`

## 人员4：借阅管理工程师

**角色定位**：负责借阅相关的核心业务，这是整个项目**技术含量最高**的模块。

### 负责文件（3个）

#### 1. `dao/BorrowRecordDao.java`
- **功能**：借阅记录数据访问层
- **核心方法**：
  - `findById(Long id)` — 查询单条记录（含联表：book标题、作者、借阅人姓名）
  - `findByIdForUpdate(Connection conn, Long id)` — **悲观锁查询**
  - `existsActiveBorrow(Long userId, Long bookId)` — 检查是否已借（事务内用）
  - `existsActiveBorrowByBookId(Long bookId)` — 检查书籍是否有未还借阅（删除书籍用）
  - `existsActiveBorrowByUserId(Long userId)` — 检查用户是否有未还借阅（删除用户用）
  - `save(BorrowRecord record)` — 插入借阅记录
  - `save(Connection conn, BorrowRecord record)` — **事务内插入**（接收外部 Connection）
  - `update(BorrowRecord record)` — 更新记录（归还时用）
  - `update(Connection conn, BorrowRecord record)` — **事务内更新**
  - `findAll(status, userId, page, pageSize)` — 分页查询（含联表）
  - `count(status, userId)` — 统计总数
- **如何上手**：
  1. **联表查询**是重点：`LEFT JOIN books b ON br.book_id = b.id LEFT JOIN users u ON br.user_id = u.id`
  2. 注意有**两个版本**的方法：普通版（自己获取连接）和事务版（接收外部连接）
  3. count 方法的 WHERE 条件**不要加表别名**（因为没有 JOIN）

#### 2. `service/BorrowService.java`
- **功能**：借阅业务逻辑（**项目核心**，含手动事务和悲观锁）
- **核心方法**：
  - `getMyBorrowRecords(userId, status, page, pageSize)` — 我的借阅记录
  - `getAllBorrowRecords(status, userId, page, pageSize)` — 全部借阅记录
  - `borrowBook(userId, bookId)` — **借书（含事务）**
  - `returnBook(userId, recordId, userRole)` — **还书（含事务）**
- **借书事务流程**：
  ```
  conn = DatabaseUtil.getConnection()
  conn.setAutoCommit(false)           // 开启事务
  try {
      book = bookDao.findByIdForUpdate(conn, bookId)   // 锁定书籍行
      检查库存 > 0
      检查未借过该书
      borrowRecordDao.save(conn, record)                // 插入借阅记录
      bookDao.decreaseAvailable(conn, bookId)           // 扣减库存
      conn.commit()                                     // 提交
  } catch (Exception e) {
      conn.rollback()                                   // 回滚
      throw e;
  } finally {
      conn.close()
  }
  ```
- **如何上手**：
  1. **这是最难的部分**，必须理解事务的 ACID
  2. `FOR UPDATE` 锁定行后，其他线程访问同一行会阻塞，直到事务结束
  3. rollback 在 catch 中执行，确保异常时数据回滚
  4. finally 中必须关闭连接，否则连接泄漏

#### 3. `console/BorrowMenu.java`
- **功能**：借阅管理的控制台菜单
- **核心方法**：
  - `showMenu(userId, isAdmin)` — 主菜单路由
  - `borrowBook(userId)` — 借书交互（显示书籍信息+确认）
  - `returnBook(userId, isAdmin)` — 还书交互（显示借阅信息+确认）
  - `listMyRecords(userId)` — 我的借阅记录
  - `listAllRecords()` — 全部借阅记录（管理员）
  - `listRecords(userId, isAdmin)` — 通用记录列表（含状态筛选）
- **如何上手**：
  1. 借书前先显示书籍详情让用户确认
  2. 还书前先显示借阅信息让用户确认
  3. 状态筛选：输入 "borrowing" 筛选借阅中，"returned" 筛选已归还，回车显示全部
  4. 表格列：ID、书籍、借阅人、借阅日期、应还日期、状态

## 人员5：控制台入口工程师

**角色定位**：负责程序入口和公共工具，是用户看到的第一个界面。

### 负责文件（3个）

#### 1. `Main.java`
- **功能**：程序入口，菜单路由，全局状态管理
- **核心逻辑**：
  - 初始化数据库连接池（DatabaseUtil.init()）
  - 初始化所有 Service 和 Menu 对象
  - 主循环：清屏 → 显示菜单 → 读取选择 → 路由到对应功能
  - 全局状态：currentUserId, currentUsername, currentRole
- **菜单路由逻辑**：
  ```
  未登录：1=登录, 2=注册, 0=退出
  普通用户：1=书籍浏览, 2=借阅管理, 0=退出登录
  管理员：1=书籍管理, 2=借阅管理, 3=用户管理, 4=退出登录
  ```
- **如何上手**：
  1. switch 表达式路由：`case 1 -> bookMenu.showMenu(isAdmin)`
  2. 登录成功后设置全局状态：currentUserId = user.get("id")
  3. 退出登录时清空状态
  4. `AnsiConsole.systemInstall()` 必须在 main 方法最开始调用

#### 2. `console/ConsoleHelper.java`
- **功能**：终端工具类，整个项目的 UI 基础设施
- **核心方法**：
  - `clearScreen()` — 清屏（ANSI 转义码）
  - `printTitle(title)` — 打印带边框的标题
  - `printMenuLine(text)` — 打印菜单行
  - `printMenuEnd()` — 打印菜单底边
  - `printSuccess/Error/Info(msg)` — 彩色提示信息
  - `printTable(headers, rows, widths)` — **打印表格**（最复杂的方法）
  - `readInt(prompt)` / `readInt(prompt, min, max)` — 读取数字（含校验）
  - `readString(prompt)` / `readString(prompt, required)` — 读取字符串
  - `waitForKey()` — 等待回车（含 EOF 处理）
- **表格打印逻辑**：
  ```
  ┌──────┬──────────┬──────────┐    ← 顶部边框
  │ ID   │ 书名     │ 作者     │    ← 表头（加粗）
  ├──────┼──────────┼──────────┤    ← 分隔线
  │  1   │ Java     │ 张三     │    ← 数据行
  └──────┴──────────┴──────────┘    ← 底部边框
  ```
- **如何上手**：
  1. 所有颜色用 `Ansi.ansi().fgCyan().a("文本").reset()`
  2. 表格列宽用 `padRight()` 方法对齐
  3. `readInt` 要处理非数字输入（循环重试）
  4. `waitForKey` 要处理 EOF（管道输入结束）

#### 3. `console/UserMenu.java`
- **功能**：用户管理菜单（管理员用）
- **核心方法**：
  - `showMenu(currentUserId)` — 主菜单路由
  - `listUsers(isSearch)` — 用户列表（含搜索）
  - `showUserDetail()` — 查看用户详情
  - `updateUser(currentUserId)` — 编辑用户
  - `deleteUser()` — 删除用户（含确认）
- **如何上手**：
  1. 与 BookMenu 结构类似，可以参考
  2. 编辑用户时显示当前值，回车跳过不修改
  3. 角色显示："admin" → "管理员"，"user" → "普通用户"

## 人员6：异常与配置工程师

**角色定位**：负责项目的异常处理体系和构建配置。

### 负责文件（5个）

#### 1. `exception/BusinessException.java`
- **功能**：自定义业务异常类
- **内容**：继承 RuntimeException，包含 code(int) 和 message(String)
- **两个构造方法**：
  - `BusinessException(ErrorCode errorCode)` — 通过枚举构造
  - `BusinessException(int code, String message)` — 直接指定错误码和消息


- **功能**：Maven 项目配置，声明所有依赖
- **关键依赖**：

- **如何上手**：
  1. 复制现有 pom.xml 结构，确保依赖版本正确
  2. shade 插件需要排除签名文件（META-INF/*.SF, *.DSA, *.RSA）
  3. 配置 mainClass 为 `com.library.Main`
