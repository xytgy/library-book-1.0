# 6人团队分工计划（详细版）

---

## 人员1：数据库与模型工程师

**角色定位**：负责整个项目的数据基础，包括数据库设计和所有实体类。

### 负责文件（4个）

#### 1. `database/init.sql`
- **功能**：数据库初始化脚本，创建所有表并插入测试数据
- **内容**：
  - 创建 `users` 表（id, username, password, name, email, role, status, created_at, updated_at）
  - 创建 `books` 表（id, isbn, title, author, category, location, total_quantity, available_quantity, description, created_at, updated_at）
  - 创建 `borrow_records` 表（id, user_id, book_id, borrow_date, due_date, return_date, status, created_at, updated_at）
  - 外键约束：borrow_records.user_id → users.id，borrow_records.book_id → books.id
  - 索引：user_id、book_id、status
  - 测试数据：admin/user1 账号 + 5本书
- **如何上手**：
  1. 先理解三张表的关系：用户借书 → 产生借阅记录
  2. 注意字段类型：password 用 VARCHAR(255)（BCrypt哈希很长），role 用 ENUM
  3. 密码哈希格式必须是 `$2a$10$` 开头（BCrypt 标准）
  4. 执行命令测试：`mysql -u root -p123456 < database/init.sql`

#### 2. `model/User.java`
- **功能**：用户实体类，对应 users 表
- **字段**：id(Long), username(String), password(String), name(String), email(String), role(String), status(Integer), createdAt(LocalDateTime), updatedAt(LocalDateTime)
- **如何上手**：提供无参构造方法 + 所有 getter/setter

#### 3. `model/Book.java`
- **功能**：书籍实体类，对应 books 表
- **字段**：id, isbn, title, author, category, location, totalQuantity, availableQuantity, description, createdAt, updatedAt

#### 4. `model/BorrowRecord.java`
- **功能**：借阅记录实体类，对应 borrow_records 表
- **字段**：id, userId, bookId, borrowDate, dueDate, returnDate, status, createdAt, updatedAt
- **额外字段**（联表查询用）：bookTitle(String), bookAuthor(String), userName(String)

## 人员2：用户管理工程师

**角色定位**：负责用户相关的完整功能链路：数据访问 → 业务逻辑 → 界面交互。

### 负责文件（4个）

#### 1. `dao/UserDao.java`
- **功能**：用户数据访问层
- **核心方法**：findById, findByUsername, existsByUsername, save, update, deleteById, findAll, count
- **如何上手**：每个方法模板：获取连接 → 预编译SQL → 设置参数 → 执行 → 处理结果 → 关闭资源

#### 2. `service/UserService.java`
- **功能**：用户业务逻辑层
- **核心方法**：getUsers, getUserById, updateUser（含权限校验）, deleteUser（检查借阅记录）
- **审计日志**：更新/删除用户后记录日志

#### 3. `console/AuthMenu.java`
- **功能**：登录和注册的控制台交互
- **核心方法**：showLoginMenu, showRegisterMenu

#### 4. `util/JwtUtil.java`
- **功能**：JWT 令牌工具类
- **核心方法**：generateToken, parseToken, validateToken, getUserId, getUsername, getRole

## 人员3：书籍管理工程师

**角色定位**：负责书籍相关的完整功能链路。

### 负责文件（3个）

#### 1. `dao/BookDao.java`
- **功能**：书籍数据访问层
- **核心方法**：findById, findByIdForUpdate（悲观锁）, existsByIsbn, save, update, deleteById, decreaseAvailable, increaseAvailable, findAll, count
- **如何上手**：`findByIdForUpdate` 是并发安全的关键，SQL 加 `FOR UPDATE` 后缀

#### 2. `service/BookService.java`
- **功能**：书籍业务逻辑
- **核心方法**：getBooks, getBookById, createBook（ISBN校验）, updateBook（库存联动）, deleteBook（检查借阅）
- **审计日志**：增删改书籍后记录日志

#### 3. `console/BookMenu.java`
- **功能**：书籍管理的控制台菜单
- **核心方法**：showMenu, listBooks, showBookDetail, createBook, updateBook, deleteBook

## 人员4：借阅管理工程师

**角色定位**：负责借阅相关的核心业务，整个项目**技术含量最高**的模块。

### 负责文件（3个）

#### 1. `dao/BorrowRecordDao.java`
- **功能**：借阅记录数据访问层
- **核心方法**：findById, findByIdForUpdate（悲观锁）, existsActiveBorrow, existsActiveBorrowByBookId, existsActiveBorrowByUserId, save（普通+事务版）, update（普通+事务版）, findAll（联表查询）, count
- **如何上手**：注意有两个版本的方法：普通版和事务版（接收外部 Connection）

#### 2. `service/BorrowService.java`
- **功能**：借阅业务逻辑（**项目核心**，含手动事务和悲观锁）
- **核心方法**：getMyBorrowRecords, getAllBorrowRecords, borrowBook（含事务）, returnBook（含事务）
- **审计日志**：借书/还书成功后记录日志
- **事务流程**：setAutoCommit(false) → FOR UPDATE → 操作 → commit → rollback on error

#### 3. `console/BorrowMenu.java`
- **功能**：借阅管理的控制台菜单
- **核心方法**：showMenu, borrowBook, returnBook, listMyRecords, listAllRecords

## 人员5：控制台入口工程师

**角色定位**：负责程序入口和公共工具，是用户看到的第一个界面。

### 负责文件（3个）

#### 1. `Main.java`
- **功能**：程序入口，菜单路由，全局状态管理
- **菜单路由**：未登录(登录/注册/退出) → 普通用户(书籍/借阅/退出) → 管理员(书籍/借阅/用户/审计日志/退出)
- **如何上手**：switch 表达式路由，AnsiConsole.systemInstall() 必须在最开始调用

#### 2. `console/ConsoleHelper.java`
- **功能**：终端工具类，整个项目的 UI 基础设施
- **核心方法**：clearScreen, printTitle, printMenuLine, printMenuEnd, printSuccess/Error/Info, printTable, readInt, readString, waitForKey
- **如何上手**：所有颜色用 Ansi.ansi().fgCyan().a("文本").reset()，表格列宽用 padRight() 对齐

#### 3. `console/UserMenu.java`
- **功能**：用户管理菜单（管理员用）
- **核心方法**：showMenu, listUsers, showUserDetail, updateUser, deleteUser

## 人员6：异常、配置与审计日志工程师

**角色定位**：负责项目的异常处理体系、构建配置、以及审计日志模块。

### 负责文件（7个）

#### 1. `exception/BusinessException.java`
- **功能**：自定义业务异常类
- **内容**：继承 RuntimeException，包含 code(int) 和 message(String)

#### 2. `exception/ErrorCode.java`
- **功能**：错误码枚举，统一管理所有错误码（401/403/404/400/500）

#### 3. `pom.xml`
- **功能**：Maven 项目配置，声明所有依赖
- **关键依赖**：mysql-connector-j, HikariCP, jjwt, jansi, bcrypt, slf4j+logback

#### 4. `application.properties`
- **功能**：应用配置文件（数据库、连接池、JWT）

#### 5. `logback.xml`
- **功能**：日志配置文件

#### 6. `util/AuditLogger.java`（163行）
- **功能**：审计日志工具类，记录操作到文件
- **核心方法**：
  - `log(username, action, detail)` — 记录日志
  - `readAllLogs()` — 读取全部日志
  - `searchLogs(keyword)` — 关键词搜索
  - `searchLogsByUser(username)` — 按用户搜索
  - `searchLogsByDate(date)` — 按日期搜索
  - `searchLogsByAction(action)` — 按操作类型搜索
  - `getStats()` — 统计分析
  - `exportLogs(path)` — 导出日志
  - `cleanOldLogs(days)` — 清理旧日志
- **如何上手**：写入 `logs/audit.log`，用 BufferedWriter 追加写入

#### 7. `console/AuditMenu.java`（138行）
- **功能**：审计日志管理菜单（管理员专用）
- **核心方法**：showMenu, viewAllLogs, searchByUser, searchByDate, searchByAction, showStats, exportLogs
- **如何上手**：分页浏览日志，支持多种搜索方式
