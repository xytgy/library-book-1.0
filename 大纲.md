# 图书馆管理系统 — 设计大纲

## 一、系统概述

基于 Java SE + JDBC + MySQL 的控制台图书馆管理系统，实现图书借阅、归还、查询等核心功能。

### 技术栈
- **语言**: Java 17（纯 Java SE，无框架）
- **数据库**: MySQL + HikariCP 连接池
- **认证**: JWT（jjwt）+ BCrypt 密码加密
- **终端**: jansi 彩色输出

---

## 二、用户认证模块

### 2.1 登录
- 用户输入用户名和密码
- 通过 BCrypt 验证密码
- 验证成功生成 JWT Token
- 验证失败提示"用户名或密码错误"

### 2.2 注册
- 新用户填写注册信息（用户名、密码、姓名、邮箱）
- 验证用户名唯一性
- 密码通过 BCrypt 加密存储
- 注册成功自动登录

### 2.3 用户角色
- **普通用户**: 借书、还书、查询书籍、查看个人借阅记录
- **管理员**: 所有普通用户功能 + 书籍管理、用户管理、查看全部借阅记录、审计日志

---

## 三、功能模块

### 3.1 书籍管理（管理员）
- 添加新书籍（ISBN、书名、作者、分类、位置、库存数量）
- 编辑书籍信息（联动调整可借数量）
- 删除书籍（需检查是否有未归还借阅）
- 查看书籍列表（支持分页、搜索）

### 3.2 借书功能
- 选择书籍 ID 进行借阅
- 使用 SELECT FOR UPDATE 悲观锁确保并发安全
- 检查库存 > 0
- 检查用户未借阅该书
- 创建借阅记录 + 扣减库存（同一事务）

### 3.3 还书功能
- 输入借阅记录 ID 进行归还
- 使用 SELECT FOR UPDATE 悲观锁确保并发安全
- 验证归还权限（本人或管理员）
- 更新借阅记录 + 增加库存（同一事务）

### 3.4 书籍查询
- 按书名、作者、ISBN 模糊搜索
- 按分类筛选
- 查看书籍详情

### 3.5 用户管理（管理员）
- 查看用户列表（支持搜索、分页）
- 编辑用户信息（姓名、邮箱、角色、状态）
- 删除用户（需检查是否有未归还借阅）

### 3.6 借阅记录
- 查看当前借阅（普通用户仅看自己的）
- 查看全部借阅记录（管理员）
- 按状态筛选（borrowing/returned）

### 3.7 审计日志（管理员）
- 记录所有关键操作（借书、还书、增删改书籍、增删改用户）
- 查看全部日志（分页浏览）
- 按用户/日期/操作类型搜索
- 统计分析（各操作类型次数）
- 导出日志到文件
- 日志存储在 `logs/audit.log`

## 五、并发安全设计

### 5.1 借书操作
```
事务开始
  → SELECT * FROM books WHERE id = ? FOR UPDATE  (锁定书籍行)
  → 检查 available_quantity > 0
  → 检查不存在 active borrow 记录
  → INSERT borrow_records
  → UPDATE books SET available_quantity = available_quantity - 1
事务提交
```

### 5.2 还书操作
```
事务开始
  → SELECT * FROM borrow_records WHERE id = ? FOR UPDATE  (锁定借阅记录行)
  → 检查状态不是 "returned"
  → 检查归还权限
  → UPDATE borrow_records SET status = 'returned', return_date = now()
  → UPDATE books SET available_quantity = available_quantity + 1
事务提交
```

## 六、审计日志设计

### 6.1 日志格式
```
[2026-06-28 15:30:22] [INFO] 用户: admin | 操作: 借书 | 详情: 书籍ID:5
```

### 6.2 记录的操作类型
| 操作 | 触发时机 |
|------|----------|
| 借书 | 借阅成功后 |
| 还书 | 归还成功后 |
| 添加书籍 | 书籍创建成功后 |
| 更新书籍 | 书籍修改成功后 |
| 删除书籍 | 书籍删除成功后 |
| 更新用户 | 用户信息修改成功后 |
| 删除用户 | 用户删除成功后 |

### 6.3 管理员功能
- 查看全部日志（分页）
- 按用户名搜索
- 按日期搜索
- 按操作类型搜索
- 统计分析（各操作次数）
- 导出日志到文件
