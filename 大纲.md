# 图书馆管理系统

## 一、系统概述

基于Web的图书馆管理系统，实现图书借阅、归还、查询等核心功能。

### 技术栈
- **前端**: Vue.js 3 + Element Plus
- **后端**: Java 17 + Spring Boot 3.2 + JDBC
- **数据库**: MySQL

---

## 二、用户认证模块

### 2.1 登录
- 用户输入账号密码
- 验证成功跳转到管理页面
- 验证失败提示错误信息
- 提供"去注册"链接

### 2.2 注册
- 新用户填写注册信息（用户名、密码、确认密码、姓名、邮箱）
- 验证用户名唯一性
- 密码加密存储
- 注册成功自动登录或跳转到登录页

### 2.3 用户角色
- **普通用户**: 借书、还书、查询书籍
- **管理员**: 所有普通用户功能 + 书籍管理、用户管理、借阅记录查看

---

## 三、功能模块

### 3.1 书籍管理（管理员）
- 添加新书籍（ISBN、书名、作者、分类、位置、库存数量）
- 编辑书籍信息
- 删除书籍
- 查看书籍列表（支持分页、搜索、筛选）

### 3.2 借书功能
- 搜索/选择要借阅的书籍
- 检查书籍库存
- 创建借阅记录
- 扣减书籍库存
- 显示借阅期限（默认30天）

### 3.3 还书功能
- 查看当前借阅列表
- 选择要归还的书籍
- 更新借阅记录（归还日期）
- 增加书籍库存
- 计算逾期罚款（可选）

### 3.4 书籍查询
- 按书名、作者、ISBN搜索
- 查看书籍详情（位置、库存、状态）
- 显示借阅状态（可借/已借出/预计归还日期）

### 3.5 用户管理（管理员）
- 查看用户列表
- 编辑用户信息
- 禁用/启用用户
- 查看用户借阅历史

### 3.6 借阅记录
- 查看当前借阅
- 查看历史借阅记录
- 筛选（按用户、按书籍、按日期）

## 四、数据库设计

### 4.1 用户表 (users)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 用户ID | 主键，自增 |
| username | VARCHAR(50) | 用户名 | 唯一，非空 |
| password | VARCHAR(255) | 密码（加密） | 非空 |
| name | VARCHAR(50) | 姓名 | 非空 |
| email | VARCHAR(100) | 邮箱 | |
| role | ENUM | 角色 | user/admin，默认user |
| status | TINYINT | 状态 | 1正常/0禁用，默认1 |
| created_at | DATETIME | 创建时间 | |
| updated_at | DATETIME | 更新时间 | |

### 4.2 书籍表 (books)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 书籍ID | 主键，自增 |
| isbn | VARCHAR(20) | ISBN号 | 唯一 |
| title | VARCHAR(200) | 书名 | 非空 |
| author | VARCHAR(100) | 作者 | 非空 |
| category | VARCHAR(50) | 分类 | |
| location | VARCHAR(50) | 位置（如A-01-01） | |
| total_quantity | INT | 总库存 | 默认0 |
| available_quantity | INT | 可借数量 | 默认0 |
| description | TEXT | 简介 | |
| created_at | DATETIME | 创建时间 | |
| updated_at | DATETIME | 更新时间 | |

### 4.3 借阅记录表 (borrow_records)
| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | INT | 记录ID | 主键，自增 |
| user_id | INT | 用户ID | 外键 |
| book_id | INT | 书籍ID | 外键 |
| borrow_date | DATETIME | 借阅日期 | |
| due_date | DATETIME | 应还日期 | |
| return_date | DATETIME | 实际归还日期 | 空表示未归还 |
| status | ENUM | 状态 | borrowing/returned |
| created_at | DATETIME | 创建时间 | |
| updated_at | DATETIME | 更新时间 | |

## 六、API接口设计

### 认证接口
- POST /api/auth/register - 用户注册
- POST /api/auth/login - 用户登录

### 书籍接口
- GET /api/books - 获取书籍列表
- GET /api/books/:id - 获取书籍详情
- POST /api/books - 添加书籍（管理员）
- PUT /api/books/:id - 更新书籍（管理员）
- DELETE /api/books/:id - 删除书籍（管理员）

### 借阅接口
- GET /api/borrow/my - 我的借阅记录
- POST /api/borrow/:bookId - 借书
- POST /api/borrow/return/:recordId - 还书
- GET /api/borrow/all - 所有借阅记录（管理员）

### 用户接口
- GET /api/users - 用户列表（管理员）
- GET /api/users/:id - 用户详情
- PUT /api/users/:id - 更新用户
- DELETE /api/users/:id - 删除用户（管理员）
